## TL;DR for the PM

The reviewer-and-handoff half of Epic 4 is in. A primed backlog now drains through: claim → dev subagent → push + PR → reviewer subagent → inline comments + verdict → labels → version-stamped re-runs. Three smokes in a row (4.6, 4.6b, 4.8) drove the full chain with only `execa` stubbed. That is the engineering heart beating.

**The single biggest takeaway is one we already knew and now have proof for:** prose-as-state-transport keeps failing; deterministic tool-layer seams keep winning. Story 4.6 burned **seven** smoke trials chasing reviewer-verdict drift in chat output, then rev-2 added a `reviewer-result.json` write and the next smoke went green in **one** trial. The lesson generalises: every load-bearing decision that currently lives in LLM prose is a future smoke failure waiting to happen.

**Three things to fix before the epic closes:**

1. **Smoke-harness setup friction is now the dominant tax on user-surface stories.** 4.6 lost 7 trials to setup drift (persona frontmatter, `hired_at`/`catalogue_version`, missing H2s, no PR URL, no remote, missing `standards.md`, grammar drift). The wrapper-skill follow-up has been called "overdue" twice. It needs a story.
2. **Worktree contract is on the honour system.** PR #138's dev worked on `main` (8 modified, 6 new untracked) with no enforcement. Locked-phrase grammar has a parser; the worktree contract doesn't. Asymmetry should close — most likely as an AC on 4-11 or a small standalone.
3. **`session-quota-exhausted` is an untyped failure class.** PR #138's run was bricked by Claude account session limit mid-dev; the output literally said *"You've hit your session limit"* and fell through as handoff-grammar drift. On a clean install with no Jack-acting-as-operator in the loop, this would brick the plugin's main promise. Needs its own typed error class with a documented operator-recovery path.

Everything else is smaller, captured below.

## What worked

### Deterministic seams keep paying off, repeatedly
- **4.6 rev-2** (the headline). Swapped chat-scraping for `reviewer-result.json` written by `runReviewerSession`. Smoke went from 7 trials to 1. Jack's note in PR #109: *"The MCP tool layer was solid; the LLM-prose-as-verdict-transport was fundamentally fragile."*
- **4.8b PR-URL extraction.** Moved from prose-scrape to a parser with anchored regex. Carried over a Medium Info-tier from 4.6 (unanchored regex) and closed it.
- **4.7 verdict version stamping + footer markers.** Re-runs are idempotent because the footer is a tool-written marker, not a prose pattern.
- **4.10 `computeAgreement`** helper landed as a pure tool, ready to be the seam under 4-10b's auto-merge gate.

The `feedback_default_to_deterministic_seams` memory entry has three fresh evidence points this epic. The principle is no longer aspirational — it's the architecture.

### Step 5 spec-validation gate paid for itself again
PR #122 (4-8b) caught a wrong Dev Note before it became a TS compile error. Same shape as 3.7 last epic. The gate has now justified itself in two consecutive epics — it stays.

### Operator-smoke harness ran three stories end-to-end with only `execa` stubbed
4-6, 4-6b, 4-8 all drove `runReviewerSession → postReviewerComments → processReviewerTranscript` with no prompt-level surprises. When the harness works, the loop works.

### Pre-authored spec shortcut
PR #112 (4-6b) skipped a full Opus authoring pass by pulling a spec already on disk. Useful pattern for "b" follow-up stories where the shape is known.

### Tail of Epic 3 closed cleanly
3-8 (BMad adapter real-world leniency) and 3-9 (LLM fallback for drifted stories) both shipped. The BMad adapter is now resilient enough to point at real-world drifted backlogs without bouncing.

## What hurt

### Smoke-harness setup friction is the new dev-loop tax
Epic 3's takeaway was *"the dev loop is product code now"*. Epic 4's update: *the smoke-harness setup is the dev loop's friction surface*. Story 4.6 burned **7 trials before clean signal**, every failure a different missing piece of operator state:

- persona file missing frontmatter
- persona missing `hired_at` / `catalogue_version`
- persona missing required H2s
- no PR URL in fake repo
- no git remote in fake repo
- no `standards.md`
- locked-phrase grammar drift in dev/reviewer output

4-8 hit it again. The wrapper skill (`project_smoke_harness_wrapper` memory) is now overdue — it was called *"overdue"* on PR #109 and *"urgent before the next user-surface story"* on the rev-2 addendum. Should be the next story authored.

### Structural-anchor test coverage lags new SKILL.md prose
4-8 (#119) and 4-6b (#112) both shipped Lows about new step prose that has no anchor in `start-skill-content.test.ts`. The pattern: "Prose can drift undetected." Locked-phrase grammar drift now has three fresh examples (4.6, 4.8, 4.10b). Either the anchor-test gets extended each story, or the locked phrases move out of prose into config.

### Spec/epic authoring leaves load-bearing gaps
- **#122 (4-8b)** — missing epic entry for the "b" follow-up; spec Dev Note claimed a destructure already existed when it didn't.
- **#129 (3-8)** — spec tagged `substrate` but had a `(user-surface)` AC, which would have routed it through the wrong review-pass budget if the resolver hadn't improvised.

Pattern: the resolvers are silently papering over spec/epic mismatches that the planning-discipline gate should catch. Worth a discipline-gate AC: spec `complexity` tag and AC tags must agree.

### Reviewer ships Info/Low residuals consistently, and one of them was a real bug carried across two PRs
Every retro this epic showed 1–5 unresolved Info/Low findings shipped anyway. Most cosmetic, but #109's *PR-URL regex not anchored* (Medium) was carried for two stories before #122 closed it. Worth a rule: Medium-or-higher findings cannot ship as residuals without an explicit override.

### Worktree contract is unenforced
PR #138's dev worked directly on `main` — 8 modified + 6 new untracked files when Jack inspected. The locked-phrase grammar has a parser; the worktree contract is honour-system. Jack's line: *"That asymmetry should close."* Yes.

### Quota exhaustion bricked a run with no typed failure class
PR #138 mid-dev: Claude account-session limit hit. Output: *"You've hit your session limit."* The dev subagent fell through as handoff-grammar drift. Jack had to manually commit/push/PR/clear `blocked_by`/bounce a spuriously-claimed 4-11 to recover. On a clean install with no Jack-acting-as-operator in the loop, this would brick the loop. Needs `session-quota-exhausted` as a typed class with an operator recovery doc.

### Breadcrumb tasks left the tree red at the commit boundary
Same PR #138: dev did primary work first, hit quota, left 4 follow-on edits undone → suite red across 5 files. Suggests either "green at every commit" rule or breadcrumb-first sequencing.

### Cross-PR comment leak (#109 trial 6)
A fake reviewer-target URL in a smoke persona happened to be a *real merged PR* (#108), and the reviewer subagent posted a comment on it. Deleted via graphql, but the sanity check — *reviewer can only `gh pr comment` against the cycle's own PR* — is still open. Low likelihood, high blast radius if it lands on a customer repo.

## Strategic state of the product

**What a non-engineer can do today (after Epic 4 so far):**

- All of Epic 3's planning surface, plus:
- `/crew:start` claims a ready story, spawns a dev subagent in a worktree, hands off to a reviewer subagent, posts inline review comments with a stamped verdict, applies labels, and (if approved) calls `completeStory`.
- Re-runs are idempotent — the version-stamped footer marker prevents duplicate verdicts.
- BMad-shaped backlogs with drifted stories no longer bounce; the adapter is lenient and falls back to an LLM extractor.

**What still doesn't work end-to-end:**

- **No auto-merge gate.** 4-10b is the user-facing closer — pause on Medium/High risk, auto-merge on Low, with override. Until it ships, the loop hands off to the operator at the verdict step.
- **No yield protocol.** 4-11 — domains outside the dev subagent's range need a locked-phrase yield to a domain owner. Today the dev plows on or fails opaque.
- **No runtime limits / telemetry.** 4-12 — without per-invocation soft/hard caps, a runaway dev subagent can quietly burn the user's quota (see PR #138).
- **`session-quota-exhausted`** isn't typed, so the "walk away, come back to merged PRs" promise has a known failure mode that today silently bricks the loop.

The "walk away" half is *almost* there. Three stories away — and one of them (4-12) directly addresses the bricking failure we just saw.

## Closing note

Epic 3's retro said *"the dev loop is product code now"*. Epic 4 has shown that **the LLM-prose-to-tool-state interface is the most fragile surface in the system, and every fragility we have absorbed by adding a deterministic seam has stayed fixed**. The remaining three stories of the epic (4-10b, 4-11, 4-12) all live on exactly this seam line. Author them with that as the bar — *"if this lives in prose, can we move it into a tool?"* — and the epic closes strong.

If there's one story to add ahead of the planned roster, it's the smoke-harness wrapper. Every user-surface story between now and the end of Epic 7 will pay tax we already know how to remove.
