## What went well

- **Six of twelve stories shipped silently.** 1.1 through 1.5 (scaffold, workspace resolver, stale-config detector, standards lookup, permission allowlist, telemetry pipeline) all approved on first review pass with no halts and no hotfixes. The shape of these stories: pure substrate, no user-facing surface, tight vitest fixture suites, typed-error precedent reused verbatim. **The smoothness corroborates the diagnosis from 1.8: when there's no user surface, document-driven verification is sufficient.**
- **Durable patterns set in early stories propagated without drift.** The typed-error class precedent from 1.2 was reused in 1.2b, 1.3, 1.4 without restating. The "three structural seams" pattern (1.4 `writeManagedFile` + 1.5 `logTelemetryEvent`/`gitCommit` + 1.6 `moveBetweenStates`) plus static-guard tests locked canonical-state mutation as a CI-enforced contract.
- **The 1.6 chaos test** — 1,000-iteration deterministic-seed PRNG run of the atomic-rename primitive — is now the operational definition of NFR8 and the cornerstone of the state machine.
- **Spec validation is earning its keep.** Three stories (1.6, 1.8, 1.10) had `spec_validation_failed` events caught *before* dev started. In each case the validator caught an example-vs-rule contradiction that would have wasted a full implementation pass.

## What hurt

Three connected incidents tell one story:

1. **Story 1.7 shipped 4/4 green ACs and was completely broken.** `/plugin install plugins/crew` is not a valid Claude Code command (the syntax is `name@marketplace`, not a path). `plugin.json` had `"skills": []` so `/crew:status` never appeared in tab-complete. No AC caught either bug because every gate read documents, not the running product. Required hotfix Story 1.7a.
2. **Story 1.7a still missed reality.** Even with corrected commands, the README's "Expected confirmation" lines were fiction — written by an agent that never ran the commands. Required Story 1.10.
3. **PR #61 surfaced eight bugs from one root cause:** `dist/` was gitignored. Jack's stale local `dist/` was the only reason any prior story appeared to work. Required Story 1.9 (commit `dist/` to git).

The structural fix lives in **Story 1.8** — the `user-surface` AC tag plus the `pre-pr-gate` script. It fails closed with `USER_SURFACE_UNVERIFIED` and demands either an automated end-to-end test or a `user_surface_verified` event with pasted real-Claude-Code output before a PR can open. On its first two production uses:

- **1.9's smoke run** exposed that `/crew:status` fails with `bmad adapter: detect lands in Story 3.3` — a real, parked dependency, now documented and tracked.
- **1.10's smoke run** produced the verbatim Claude Code 2.1.145 output that drove the README rewrite (single-line toast confirmations, `/reload-plugins` instead of restart, honest documentation of the `/crew:status` failure).

## Key lesson

> Document-driven verification consistently misses runtime reality.

Every Epic 1 story without a user surface shipped smoothly. Every story with one shipped broken at least once. The smoke gate is the right structural fix; we should expect to lean on it heavily through Epic 2 onward.
